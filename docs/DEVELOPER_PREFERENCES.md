# Developer Preferences & Guidelines

This document captures important preferences and guidelines for the Finance Learning Platform development.

## Emoji Usage Policy

**Preference**: **NO EMOJIS** in code, documentation, or UI unless explicitly requested.

### Policy
- **Do NOT use emojis** in:
  - Code comments
  - Documentation
  - UI text/content
  - Component labels
  - Error messages
  - Console logs
  - Commit messages (unless explicitly requested)

### Exceptions
- Only use emojis if the user **explicitly requests** them
- Do not add emojis proactively

### Current Status
- Emojis exist in some frontend components (Dashboard.tsx, celebration components, etc.)
- These should be removed when those components are refactored
- New code should not include emojis

- [ ] Remove emojis from Dashboard.tsx when refactoring
- [ ] Remove emojis from celebration components when updating
- [ ] Remove emojis from mood/journal components when updating
